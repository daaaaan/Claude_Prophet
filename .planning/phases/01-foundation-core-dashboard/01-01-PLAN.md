<objective>
Build the complete WebSocket infrastructure in an isolated `dashboard/` Go package: Hub for client management, Client with ping/pong lifecycle, Ticker that polls existing services and broadcasts JSON snapshots, event type definitions, HTTP handler for WebSocket upgrade, and route registration. Also enable SQLite WAL mode and wire the dashboard into main.go.

Purpose: This is the data pipeline foundation. Without WebSocket push, the frontend has no data source. Without WAL mode, dashboard reads would lock out bot writes. Without package isolation, a dashboard bug could crash the trading bot.

Output: A working `/ws` endpoint that broadcasts JSON snapshots every 2 seconds to any connected WebSocket client, plus a `/dashboard` route ready to serve templates (templates created in Plan 02).
</objective>

Create the `dashboard/` package directory and four Go files:

**dashboard/hub.go** -- WebSocket Hub (gorilla/websocket chat example pattern, broadcast-only):
- `Hub` struct with `clients map[*Client]bool`, `broadcast chan []byte`, `register chan *Client`, `unregister chan *Client`
- `NewHub()` constructor
- `Run()` method with select loop handling register/unregister/broadcast
- Wrap the `Run()` method body in `defer func() { if r := recover(); r != nil { log.Error("Hub panic recovered:", r) } }()` for WS-05 isolation
- When broadcasting, use non-blocking send to `client.send` with `default` case that closes the client channel and removes from map (gorilla pattern)

**dashboard/client.go** -- WebSocket Client with ping/pong lifecycle:
- Constants: `writeWait = 10s`, `pongWait = 60s`, `pingPeriod = 54s` (9/10 of pongWait), `maxMessageSize = 512`
- `Client` struct with `hub *Hub`, `conn *websocket.Conn`, `send chan []byte` (buffered, capacity 256)
- `readPump()`: defers `hub.unregister <- c` and `conn.Close()`, sets read limit to maxMessageSize, sets read deadline to pongWait, sets pong handler to reset read deadline, loops reading messages (loop exists only to detect disconnect -- dashboard is broadcast-only, no client-to-server messages)
- `writePump()`: creates pingPeriod ticker, defers `ticker.Stop()` and `conn.Close()`, select loop writes messages from `send` channel (with writeWait deadline) or sends ping on ticker tick

**dashboard/events.go** -- Event type definitions:
- `DashboardSnapshot` struct: `Type string "json:\"type\""`, `Timestamp time.Time`, `Account *AccountData`, `Positions []*PositionData`, `Activity []ActivityItem`, `BotHealth *BotHealthData`
- `AccountData` struct: `Cash float64`, `Equity float64` (maps from PortfolioValue), `BuyingPower float64` -- all with json tags
- `PositionData` struct: `Symbol string`, `Qty float64`, `EntryPrice float64` (from AvgEntryPrice), `CurrentPrice float64`, `UnrealizedPL float64`, `UnrealizedPLPct float64`, `MarketValue float64`, `Side string` -- all with json tags
- `ActivityItem` struct: `Timestamp time.Time`, `Type string`, `Action string`, `Symbol string`, `Details map[string]interface{}` -- maps from services.Activity
- `BotHealthData` struct: `Alive bool`, `LastActivity time.Time`, `Uptime string`

**dashboard/ticker.go** -- Polls existing services and publishes snapshots:
- `Ticker` struct with `hub *Hub`, `tradingService interfaces.TradingService`, `activityLogger *services.ActivityLogger`, `interval time.Duration`, `startTime time.Time`
- `NewTicker(hub, tradingService, activityLogger, interval)` constructor that records `startTime: time.Now()`
- `Run(ctx context.Context)` method: creates time.Ticker, defers Stop(), select loop on ctx.Done() or ticker.C calling `publishSnapshot(ctx)`. Wrap entire body in `defer func() { if r := recover(); r != nil { log.Error("Ticker panic recovered:", r) } }()` for WS-05 isolation
- `publishSnapshot(ctx)`: calls `tradingService.GetAccount(ctx)` and `tradingService.GetPositions(ctx)` and `activityLogger.GetCurrentLog()`. Convert `interfaces.Account` to `AccountData` (map PortfolioValue -> Equity). Convert `[]*interfaces.Position` to `[]*PositionData` (map AvgEntryPrice -> EntryPrice, UnrealizedPLPC -> UnrealizedPLPct). Convert recent activities from `DailyActivityLog.Activities` (last 50 items, reversed for reverse-chronological) to `[]ActivityItem`. Build `BotHealthData` with `Alive: true` (if we got account data successfully), `LastActivity: time.Now()`, `Uptime: time.Since(startTime).Round(time.Second).String()`. Marshal `DashboardSnapshot{Type: "snapshot", Timestamp: time.Now(), ...}` to JSON and send to `hub.broadcast`. Log errors but do not crash on individual poll failures -- send partial snapshots with nil fields.

Run `go get github.com/gorilla/websocket@v1.5.3` to add the dependency.
